# GitHub Projects — Installation Guide

Tools integrated into this Claude Code setup. Run `install.sh` to automate everything, or follow the manual steps below.

---

## context-mode — `mksglu/context-mode`

MCP server that keeps raw tool output out of the context window (98% reduction). Adds session continuity via SQLite + FTS5.

```bash
claude mcp add context-mode -- npx -y context-mode
```

Verify after next session start:

```bash
# In Claude Code session:
/context-mode:ctx-doctor
```

Plugins to enable in `settings.json`:

```json
"enabledPlugins": { "context-mode@context-mode": true },
"extraKnownMarketplaces": {
  "context-mode": { "source": { "source": "github", "repo": "mksglu/context-mode" } }
}
```

## lightpanda — `lightpanda-io/browser`

Fast headless browser (9x less memory than Chrome). Used by the `lightpanda` skill for web scraping.

```bash
curl -L -o ~/.local/bin/lightpanda \
    https://github.com/lightpanda-io/browser/releases/download/nightly/lightpanda-x86_64-linux
chmod a+x ~/.local/bin/lightpanda
lightpanda --version
```

No MCP registration needed — the `lightpanda` skill invokes the binary directly.

## tilth — `jahala/tilth`

AST-aware code reading MCP server. Replaces Grep/Glob/Read with structural equivalents (~40% fewer tokens per answer).

```bash
# Install binary
TILTH_VER=$(curl -s https://api.github.com/repos/jahala/tilth/releases/latest \
    | python3 -c "import sys,json; print(json.load(sys.stdin)['tag_name'])")
curl -L -o ~/.local/bin/tilth \
    "https://github.com/jahala/tilth/releases/download/${TILTH_VER}/tilth-x86_64-unknown-linux-gnu"
chmod a+x ~/.local/bin/tilth

# Register with Claude Code (--edit adds hash-anchored line output for safer edits)
tilth install claude-code --edit

# Or manually:
claude mcp add tilth -- ~/.local/bin/tilth --mcp --edit
```

Requires Rust 1.75+ for building from source. Use prebuilt binary above otherwise.

## code-review-graph

Risk-scored code review and graph navigation via MCP. Builds a `.code-review-graph/` SQLite index per project.

```bash
# Install into venv
source ~/env/.venv/bin/activate
uv pip install code-review-graph

# Register globally (MCP server + hooks in settings.json + CLAUDE.md section)
code-review-graph install claude-code

# Per project (run once per repo)
cd <project>
code-review-graph build
```

The hooks in `settings.json` use the absolute venv path to work without activation:

```json
"PostToolUse": [{
  "matcher": "Edit|Write|Bash",
  "hooks": [{ "command": "git rev-parse --git-dir >/dev/null 2>&1 && $HOME/env/.venv/bin/code-review-graph update --skip-flows || true" }]
}]
```

The `dedup.py` PreToolUse hook (`~/.claude/hooks/dedup.py`) is auto-generated by `code-review-graph install claude-code` — do not write it manually.

## token-savior — `Mibayy/token-savior`

Token optimization MCP server with semantic memory, call predictions, and code intelligence tools.

```bash
# Install core + MCP extra only — skip memory-vector to avoid PyTorch/CUDA (~2GB)
source ~/env/.venv/bin/activate
uv pip install "token-savior-recall[mcp]"

# Register with Claude Code
claude mcp add token-savior -- ~/env/.venv/bin/token-savior
```

`memory-vector` extra adds semantic search over recalled tokens but pulls in PyTorch + NVIDIA CUDA stack. Skip it on dev machines unless you need semantic memory search.

## rtk — `rtk-ai/rtk`

Token-optimized CLI proxy (60-90% savings on dev operations). Commands are transparently rewritten via a Claude Code hook.

```bash
RTK_VERSION=$(curl -s https://api.github.com/repos/rtk-ai/rtk/releases/latest \
    | python3 -c "import sys,json; print(json.load(sys.stdin)['tag_name'])")
curl -L -o ~/.local/bin/rtk \
    "https://github.com/rtk-ai/rtk/releases/download/${RTK_VERSION}/rtk-x86_64-unknown-linux-gnu"
chmod a+x ~/.local/bin/rtk
rtk --version && rtk gain
```

⚠️ Name collision: `reachingforthejack/rtk` (Rust Type Kit) uses the same binary name. Verify with `rtk gain` — if it fails, you have the wrong one.

No MCP registration needed. Usage is via direct CLI calls and hook-based command rewriting.
